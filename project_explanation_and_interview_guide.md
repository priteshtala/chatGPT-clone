# 🚀 ChatGPT Clone - Project Explanation & Interview Guide

આ ફાઈલમાં આપણે આ પ્રોજેક્ટમાં જેટલું પણ કામ કર્યું છે એનું સંપૂર્ણ **Technical Explanation (ટેકનિકલ વર્ણન)** આપેલું છે. આ તમને પ્રોજેક્ટ સમજવામાં અને ઈન્ટરવ્યુમાં પૂછાઈ શકે તેવા પ્રશ્નોના જવાબો આપવામાં ખૂબ મદદરૂપ થશે.

---

## 🏗️ 1. Project Architecture (પ્રોજેક્ટનું માળખું)
આ પ્રોજેક્ટ **Flutter** માં બનાવેલો છે અને તેમાં State Management માટે **Cubit (BLoC Pattern)** નો ઉપયોગ થયો છે. ડેટા અને API કોલિંગ માટે **Dio** નો ઉપયોગ થયો છે.

### 🔑 મુખ્ય ફીચર્સ:
1. **Multiple AI Providers (Groq, Gemini, OpenAI):** યુઝર ગમે તે AI મોડેલ પસંદ કરી શકે છે.
2. **Voice Notes (Whisper API):** WhatsApp જેવું વોઇસ રેકોર્ડિંગ અને ઓડિયો પ્લેયર.
3. **Vision Support (Images):** ફોટો મોકલીને AI ને પ્રશ્નો પૂછી શકાય.
4. **Auto-Fallback & Error Handling:** જો કોઈ સર્વર ડાઉન હોય કે એરર આવે, તો એપ જાતે જ બીજું મોડેલ ચાલુ કરી દે.

## 🛠️ 2. Technical Implementation (આપણે કઈ વસ્તુ કેવી રીતે બનાવી?)

### A. Voice Notes (ઓડિયો રેકોર્ડિંગ અને ટ્રાન્સક્રિપ્શન)
- **શું વાપર્યું?** `record` પેકેજ (અવાજ રેકોર્ડ કરવા) અને `audioplayers` પેકેજ (અવાજ સાંભળવા). 
- **લોજીક:** જ્યારે યુઝર માઈક દબાવે ત્યારે `.m4a` ફોર્મેટમાં ઓડિયો રેકોર્ડ થાય છે.
- **Whisper API:** રેકોર્ડિંગ પૂરું થતાં જ આ ફાઈલ સીધી **OpenAI ના Whisper API** (અથવા Groq ના Whisper મોડેલ) ને મોકલાય છે. આ API યુઝરના અવાજને સાંભળીને તેને **Text (લખાણ)** માં કન્વર્ટ કરે છે.

### B. Streaming vs Non-Streaming API Responses
- **સ્ટ્રીમિંગ શું છે?** જ્યારે AI જવાબ વિચારી રહ્યું હોય ત્યારે આખો જવાબ એકસાથે આવવાની રાહ જોવાને બદલે, થોડો-થોડો (ટુકડે-ટુકડે) સ્ક્રીન પર ટાઇપ થતો દેખાય તેને સ્ટ્રીમિંગ (Server-Sent Events) કહેવાય.
- **લોજીક:** આપણે `Dio` ની અંદર `ResponseType.stream` નો ઉપયોગ કર્યો છે. જેવું API માંથી ડેટાનું નવું પેકેટ આવે, તરત જ આપણે `Cubit` ની મદદથી UI માં State અપડેટ કરીએ છીએ.
- **અપવાદ (Exception):** જ્યારે ફોટો (Image) મોકલવામાં આવે ત્યારે સ્ટ્રીમિંગ બંધ કરી દેવામાં આવે છે, કારણ કે અમુક Vision મોડેલ્સ સ્ટ્રીમિંગ સપોર્ટ નથી કરતા.

### C. Image Upload & Vision Models (ફોટો સ્કેનિંગ)
- **લોજીક:** જ્યારે યુઝર ફોટો સિલેક્ટ કરે, ત્યારે આપણે એ ફોટાને **Base64 String** માં કન્વર્ટ કરીએ છીએ. પછી એને `data:image/jpeg;base64,...` ફોર્મેટમાં API ને મોકલીએ છીએ.
- **Smart Logic (સ્માર્ટ લોજીક):** જો યુઝર Llama 3.3 જેવા Text-only મોડેલ પર હોય અને ફોટો સિલેક્ટ કરે, તો આપણી એપ ઑટોમૅટિક મોડેલ બદલીને **Llama 4 Scout (Vision)** કરી દે છે, જેથી એરર ના આવે.
- **History Management:** જો યુઝર ફોટો મોકલ્યા પછી ફરીથી Text-only મોડેલ વાપરે, તો એપ જાતે જ જૂની ચેટ હિસ્ટ્રીમાંથી ફોટો હટાવીને માત્ર લખાણ જ API ને મોકલે છે, જેથી **400 Bad Request** ની એરર ના આવે.

### D. Error Handling & Auto Fallback (એરર કંટ્રોલ)
- જો Groq કે Gemini નું સર્વર ડાઉન હોય, અથવા રેટ લિમિટ (Quota Exceeded - 429) આવી જાય, તો એપ ક્રેશ નથી થતી.
- `chat_cubit.dart` માં આપણે એરર પકડીએ છીએ. જો **404 (Not Found)** કે **503 (Service Unavailable)** એરર આવે, તો એપ જાતે જ Default મોડેલ (Llama 4 Scout) પર સ્વિચ થઈ જાય છે અને યુઝરને ફરીથી Try કરવા કહે છે.

## 🎯 3. Top Interview Questions & Answers (ઈન્ટરવ્યુ માટે પ્રશ્નો)

અહીં એવા પ્રશ્નો આપેલા છે જે ઇન્ટરવ્યુ લેનાર આ પ્રોજેક્ટ જોઈને તમને પૂછી શકે છે:

> **Q1: "તમે આ એપમાં State Management માટે શું વાપર્યું છે અને શા માટે?"**
> **Answer:** "મેં BLoC આર્કિટેક્ચર અંતર્ગત **Cubit** નો ઉપયોગ કર્યો છે. Cubit વાપરવાનું કારણ એ છે કે તે BLoC કરતાં થોડું સરળ છે (કારણ કે તેમાં Events ના બદલે ડાયરેક્ટ Functions કોલ થાય છે). ચેટ એપ્લિકેશનમાં જ્યારે સ્ટ્રીમિંગ ડેટા (એક એક શબ્દ) આવે છે, ત્યારે Cubit ના `emit()` ફંક્શનથી UI ને ખૂબ જ ઝડપથી અને સ્મૂથલી રિબિલ્ડ કરી શકાય છે."

> **Q2: "તમે Voice Notes કેવી રીતે ઇમ્પ્લીમેન્ટ કર્યા છે? અવાજ ટેક્સ્ટમાં કઈ રીતે ફરે છે?"**
> **Answer:** "મેં Flutter ના `record` પેકેજથી ડિવાઇસનું માઇક્રોફોન એક્સેસ કરીને લોકલ ફાઇલ (`.m4a`) માં ઓડિયો સેવ કર્યો. પછી એ ઓડિયો ફાઇલને Dio `MultipartFile` દ્વારા OpenAI ના **Whisper-1** (અથવા Groq Whisper) API ને મોકલી. Whisper મોડેલ એ ઓડિયો પ્રોસેસ કરીને ટેક્સ્ટ (String) પાછું આપે છે, જે હું ચેટ બોક્સના કંટ્રોલરમાં સેટ કરી દઉં છું."

> **Q3: "API માંથી જ્યારે ટુકડે-ટુકડે શબ્દો (Streaming) આવે છે ત્યારે એપમાં કઈ રીતે દેખાય છે?"**
> **Answer:** "ચેટ API ને કોલ કરતી વખતે મેં Dio માં `ResponseType.stream` સેટ કર્યું છે. આનાથી મને એક `Stream<List<int>>` મળે છે. હું તેને `utf8.decoder` અને `LineSplitter` નો ઉપયોગ કરીને લાઈન-બાય-લાઈન ડીકોડ કરું છું. જ્યારે ડેટા લાઈન `data: ` થી શરૂ થાય, ત્યારે હું JSON પાર્સ કરીને નવો શબ્દ જૂના લખાણમાં જોડી દઉં છું અને Cubit માં State અપડેટ કરું છું."

> **Q4: "જો કોઈ મોડેલ ઈમેજ સપોર્ટ ના કરતું હોય તો તમારી એપ તેને કઈ રીતે હેન્ડલ કરે છે?"**
> **Answer:** "આ ખૂબ જ અગત્યનો એજ-કેસ (Edge Case) હતો. જો યુઝર Llama 3.3 (Text-only) વાપરતો હોય અને ફોટો અપલોડ કરે, તો એપ ઑટોમૅટિક મોડેલ બદલીને વિઝન સપોર્ટવાળું Llama 4 Scout કરી નાખે છે. અને જો ભૂતકાળમાં ઈમેજ મોકલી હોય અને પછી યુઝર Text મોડેલ સિલેક્ટ કરે, તો મારી એપ `messages.map` કરતી વખતે જૂના મેસેજમાંથી `image_url` ને ફિલ્ટર કરી (હટાવી) દે છે, જેથી API 400 Bad Request એરર ન આપે."

> **Q5: "જો API 503 કે 429 એરર આપે તો શું થાય?"**
> **Answer:** "મેં Dio ના `DioException` દ્વારા એરર હેન્ડલિંગ કર્યું છે. જો 429 (Rate Limit) આવે તો યુઝરને 'Quota Exceeded' નો ફ્રેન્ડલી મેસેજ દેખાય છે. જો 404 કે 503 એરર આવે, તો `ChatCubit` ઑટોમૅટિક રીતે મોડેલને Default મોડેલમાં બદલી નાખે છે અને યુઝરને 'મોડેલ બદલાઈ ગયું છે, ફરીથી પ્રયાસ કરો' તેવો મેસેજ આપે છે."

> **Q6: "ફોટા (Images) ને API માં કયા ફોર્મેટમાં મોકલવામાં આવે છે?"**
> **Answer:** "મેં `image_picker` થી ફોટો સિલેક્ટ કરાવીને તેને `readAsBytes()` થી બાઈટ્સમાં ફેરવ્યો. પછી તેને `base64Encode()` કરી લીધો. છેલ્લે API ને મોકલતી વખતે તેને Data URI ફોર્મેટ એટલે કે `data:image/jpeg;base64,<Base64String>` તરીકે મોકલ્યો છે."
